    注意事项

    一、hexo g只负责用你选择的主题去渲染你的博客原文.md文件，不负责生成CNAME,即当你在github上保存好域名，生成CNAME文件后，运行hexo g；hexo d它将把生成的CNAME删除掉，导致买的域名解析失败

    • 解决方法：

    在新建仓库，不要急着将买的域名保存为自定义的域名，可以在执行hexo d前在根目录下的source里新建CNAME文 件，写上自己买的域名，因为在source目录下，只有_posts里的内容才被渲染成html在网站上显示其他文件都是不做渲染直接被推送到远程仓库中的

    若远程仓库中已有CNAME，则删除该文件，重复步骤1即可

    二、任何对主题配置文件_config.yml修改都需要在该主题目录下执行git pull，然后重新生成，部署才能让更改生效，对hexo根目录下的 _config.yml修改也得重新生成，部署才能生效

    三、被 hexo init 初始化的目录结构：

    ​ deploy_git：远程仓库的本地库

    ​ public：存放生成的静态页面，和其他普通文件/文件夹，其整个内容都与source里的内容对应

    ​ scaffolds：存放layout 布局文件，默认有三个布局文件，当然你也可以仿照已有的新建布局文件，通过hexo new [layout] “filename” 可以快速创建一定格式的md文件（用默认的就好了，省得麻烦）

    ​ source：部署到远程仓库的初始文件夹，其中_posts为博客原文文件夹

    ​ themes：存放着克隆到的hexo主题文件夹
